# Context Engineering

## File-as-Truth 原则
- 文件内容是唯一真相来源，不依赖记忆
- 修改前先读取当前文件状态
- 声称完成前必须运行验证命令

## 子代理使用
- 子代理返回结果 < 200 字
- 包含: 做了什么、结果、关键数据点
- 不传递整个文件内容给子代理

## Token 预算意识
- 交易循环 (trade-loop) 每次 ≤ $1.00 token 预算
- 扫描 (scan) 每次 ≤ $0.50
- 优先读取需要的文件片段，不读整个文件
- 长输出截断到关键部分

## 验证链 Context 优化
- regime-detect: 只需 BTC 1h K线，输出 1 行状态即可，不展开解释
- bull-bear-debate: 牛方/熊方各 3 条论点，评分后一句话结论
- reasoning-audit: 三角检查用表格形式，不写长段分析
- risk-check: 7 项检查一行一项，通过/不通过
- 扫描 50 个币的信号输出用表格，不逐个写段落
- 只对有 ⚡ 标记的异常币种做深度分析，其余跳过
